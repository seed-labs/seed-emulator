Task 4 aims to establish a trustworthy agentic orchestration methodology for closed-loop network experimentation over a high-fidelity Internet simulation environment. The problem is not merely to let a language model call tools. The scientific objective is to endow a realistic, executable network emulator with a disciplined autonomous control layer that can translate experimental intent into grounded, policy-bounded, evidence-producing action. This task therefore serves as the capstone of the broader project: the simulation substrate provides realistic Internet-scale scenarios; the runtime control and evidence plane provides structured observability, bounded intervention, and artifact collection; and the agentic layer integrates task formalization, planning, execution supervision, verification, and archival into a closed experimental loop.

At the bottom of the architecture is a high-fidelity network simulation substrate. It instantiates autonomous systems, routing relationships, services, and representative fault or attack scenarios in executable form. This layer should be understood as a research instrument for Internet and security experimentation rather than as a static model. Above it sits a runtime control and evidence plane that attaches to running simulated systems, reconstructs structured inventory, scopes observation and intervention to selected nodes or components, schedules deterministic operational jobs, and preserves logs, measurements, and verification outputs as durable artifacts. This intermediate layer is crucial because it turns the simulation substrate into an observable and controllable experimental environment.

On top of these foundations, the trustworthy agentic orchestration layer converts high-level scientific objectives into explicit, auditable workflows. It formalizes tasks with declared objectives, input requirements, admissible action boundaries, policy constraints, expected artifacts, and rollback criteria. It reasons over structured runtime state rather than ad hoc system output. It selects bounded interventions through an explicit operational surface, re-observes the environment after each step, verifies post-conditions, and archives evidence to support replay, comparison, and scientific inspection. Elevated actions are not treated as free-form generations; they are mediated by policy gates and, when necessary, human confirmation.

The architecture is inherently closed loop. Runtime observations feed into task-aware planning; planning leads to bounded interventions; interventions trigger verification and evidence production; archived evidence informs both reproducibility and subsequent reasoning. This supports rigorous workflows such as routing diagnosis, service restoration, controlled fault injection, impact analysis, rollback, and post-change verification. The figure should communicate this as a macro scientific architecture with three major layers and a strong observe-plan-act-verify-archive loop. It should feel conceptual, strategic, and methodological. It should not read like a software stack diagram, an operations dashboard, or an implementation workflow.
