# SEED 智能体展示口播提纲

## 一句话定位

今天不展示“怎么造网络”，展示的是：  
**SEED 已经跑起来以后，智能体如何进入运行时环境、理解状态、做受控动作、验证结果，并把它变成可复查的实验闭环。**

---

## 第一段：主线

开场口径：

- SEED 当然能构建复杂网络，但今天主角不是 build path。
- 今天主角是 attached-runtime，也就是网络已经跑起来之后，智能体还能做什么。

主线要点：

1. 我们先让 agent attach 到一个已经运行的邮件网络。
2. 它不是从零聊天，而是能看懂当前环境、节点角色、服务范围和下一步缺什么输入。
3. 这一步证明的是：我们已经不是“给模型几个 shell 命令”，而是让它进入一个真实运行中的实验系统。

建议收束句：

> 第一幕只证明一件事：agent 知道自己在哪，也知道该怎么继续问和继续做。

---

## 第二段：场景池

这里不要只盯着 B00/B29，要把场景池摊开。

核心话术：

- B00/B29 只是今天最成熟的样板，不是全部潜力。
- SEED 的 example 版图很大，已经覆盖：
  - Internet / DNS / Email
  - 路由安全
  - Tor / Botnet / 攻击复现
  - IPFS / Anycast / PKI
  - SCION
  - Ethereum / Chainlink / Monero

这一段要强调：

- 今天 live 只取最稳的样板
- 但同一个仿真器生态里，其实已经有很多未来可以转成 agent 场景的网络
- 这说明我们做的不是一个窄功能 demo，而是一个可扩展的方法和平台方向

建议收束句：

> 所以今天看到的不是“这一个例子”，而是一整类 future agent-ready 场景的入口。

---

## 第三段：路线图

最后一段讲后续，不讲空愿景，讲怎么做。

核心话术：

- 后续不是继续堆命令，而是把更多 example 转成标准 agent-ready 场景包。
- 每个场景包都要有同样的 contract：
  - attach 入口
  - 一句话目标
  - 可观察信号
  - 成功条件
  - 风险门
  - rollback
  - evidence 输出

后续分工可以这样讲：

- 网络运维组：把 Internet 类例子转成 maintenance 样板
- 安全演练组：把 prefix hijack / PKI / botnet 转成 security 样板
- 实验自动化组：把 traffic / latency / packet loss 转成 experiment 样板
- 新协议与平台组：把 SCION / 区块链 / IPFS 转成新型 runtime 样板
- 文档与 review 组：统一场景模板、报告模板、评审标准

建议收束句：

> 我们真正要做的，不是一个会调工具的模型，而是一套能把运行中网络实验系统逐步变成“智能实验伙伴”的方法和平台。

---

## 现场结尾三句话

1. 今天 live 只用了少数样板，是为了稳，不是因为 SEED 只有这些例子。
2. B00/B29 是已经打通闭环的样板，A/B/D/S/Y 更多系列是下一步扩展空间。
3. 我们的目标不是再写更多脚本，而是把更多网络场景系统化地转成 agent-ready 实验包。
